# CDX-E Exercise Library

This directory contains all exercise definitions. Each exercise is a self-contained
subdirectory that drives the Ansible pipeline phases defined in `site.yml`.

---

## Directory Structure

```
EXERCISES/
├── TEMPLATE/                    # Canonical scaffold — copy this to create a new exercise
│   ├── scenario.yml             # Phase gates, domain, Proxmox, APT/SOC defaults
│   ├── vms.yaml                 # Scenario VM inventory + network_topology
│   ├── exercise_template.json   # AD structure: sites, site links, OU hierarchy
│   ├── users.json               # AD user accounts and security groups
│   ├── gpo.json                 # Group Policy Objects
│   ├── services.json            # DNS zones and service configuration
│   └── README.md                # Per-exercise configuration checklist
│
└── <EXERCISE_NAME>/             # Live exercise directory
    ├── scenario.yml
    ├── vms.yaml
    ├── exercise_template.json
    ├── users.json
    ├── gpo.json
    ├── services.json
    ├── VyOS/                    # Per-router VyOS CLI config files
    │   └── <VM-NAME>.conf
    ├── terraform/               # Generated by deploy_scenario (gitignored)
    ├── terraform_red_team/      # Generated by deploy_red_team (gitignored)
    ├── terraform_blue_team/     # Generated by deploy_blue_team (gitignored)
    └── phase_state.yml          # Runtime checkpoint artifact (gitignored)
```

---

## Creating a New Exercise

```bash
# 1. Copy the template directory
cp -r EXERCISES/TEMPLATE/ EXERCISES/MY_EXERCISE_NAME/

# 2. Edit scenario.yml — set exercise name, domain, Proxmox node, phase gates
# 3. Edit vms.yaml — define your VM fleet and network topology
# 4. Edit exercise_template.json, users.json, gpo.json as needed
# 5. Add VyOS router config files to VyOS/ (one .conf per router VM)

# 6. Validate configuration parses cleanly
ansible-playbook playbooks/exercise_initiation.yml \
  -e "exercise=MY_EXERCISE_NAME" \
  -e "@secrets/credentials.yml" --ask-vault-pass

# 7. Dry-run the full pipeline (check mode)
ansible-playbook site.yml \
  -e "exercise=MY_EXERCISE_NAME" \
  -e "@secrets/credentials.yml" --ask-vault-pass \
  --check --diff
```

---

## Exercise Naming Convention

- Uppercase, underscores only: `MY_EXERCISE_NAME`
- Matches folder name exactly
- Referenced on the Ansible command line as `-e "exercise=MY_EXERCISE_NAME"`

## Runtime Artifacts (gitignored)

| Artifact | Location | Created by | Removed by |
|---|---|---|---|
| Terraform state (scenario) | `<exercise>/terraform/` | `deploy_scenario` | `destroy_scenario` |
| Terraform state (Red Team) | `<exercise>/terraform_red_team/` | `deploy_red_team` | `destroy_red_team` |
| Terraform state (Blue Team) | `<exercise>/terraform_blue_team/` | `deploy_blue_team` | `destroy_blue_team` |
| Pipeline checkpoint state | `<exercise>/phase_state.yml` | `pipeline_checkpoint` | `destroy_scenario` |
| Dynamic inventory | `inventory/exercise_hosts.yml` | `inventory_refresh` | `destroy_scenario` |
| Per-VM host_vars | `inventory/host_vars/<vm>.yml` | `vm_power_management` | `destroy_scenario` |
